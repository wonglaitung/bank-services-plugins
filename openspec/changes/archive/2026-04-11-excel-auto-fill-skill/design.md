## Context

Excel 模版填充是日常办公中常见的重复性工作。当前用户需要手动识别模版字段、复制数据、定位单元格，效率低下且易出错。本 SKILL 旨在通过智能匹配算法自动化这一流程。

**约束条件：**
- 仅支持 Excel 文件格式（.xlsx, .xls, .xlsm）
- 需要 Python 环境运行
- 作为 iFlow CLI 的 SKILL，遵循 SKILL 开发规范

**目标用户：** 需要批量填充 Excel 报表、表单的业务人员

## Goals / Non-Goals

**Goals:**
- 实现信息到 Excel 模版的智能匹配与自动填充
- 支持多种输入格式（JSON、字典、键值对文本）
- 提供精确匹配、模糊匹配、语义匹配三种匹配策略
- 支持自定义映射配置，允许用户指定字段对应关系
- 保留原模版格式（字体、边框、颜色等）

**Non-Goals:**
- 不支持跨表格公式计算
- 不支持图表、透视表等复杂对象
- 不提供在线协作功能
- 不支持数据库直接导入（可通过 JSON 中转）

## Decisions

### 1. 匹配策略选择

**决策：** 采用三层匹配策略（精确 → 模糊 → 语义）

**理由：**
- 精确匹配效率最高，优先使用
- 模糊匹配处理拼写差异、缩写等场景（使用 fuzzywuzzy）
- 语义匹配处理同义词、翻译等场景（可扩展 LLM）

**备选方案：**
- 仅精确匹配：匹配率低，用户体验差
- 直接使用 LLM：成本高、延迟大、不可控

### 2. 模版解析方式

**决策：** 解析 Excel 中带有特定标记的单元格作为字段

**标记方式：**
- `${字段名}` - 标准标记
- `{{字段名}}` - 兼容 Jinja 风格
- 无标记时，自动识别第一行/第一列作为字段名

**理由：** 标记方式灵活且不破坏模版原有内容，便于用户理解

### 3. 输入格式处理

**决策：** 统一转换为 Python 字典进行处理

**理由：**
- 字典是 Python 原生数据结构，处理效率高
- JSON、键值对文本均可方便转换为字典
- 便于后续扩展其他格式

## Risks / Trade-offs

**Risk 1: 匹配错误导致数据错位**
→ Mitigation: 提供预览功能，用户确认后执行填充；支持映射配置文件强制指定映射关系

**Risk 2: 大文件处理性能问题**
→ Mitigation: 采用流式处理，不一次性加载全部数据；设置合理的行数上限

**Risk 3: 模版格式丢失**
→ Mitigation: 使用 openpyxl 的 copy 模式，保留单元格样式；输出前校验格式完整性

**Trade-off: 语义匹配 vs 成本**
- 语义匹配更智能但可能引入额外依赖（如 LLM API）
- 初始版本优先支持精确和模糊匹配，语义匹配作为可选增强
